# PURPOSE

This document defines the complete content architecture,
information hierarchy,
navigation system,
SEO structure,
routing strategy,
future scalability,
and user flow.

The goal is to ensure that every visitor can easily understand
what JaxiCloud offers and reach their desired destination with
the fewest possible interactions.

The architecture must remain scalable for years.

------------------------------------------------------------

# CONTENT WRITING RULES

Use short paragraphs.

Avoid technical jargon where possible.

Explain features with benefits.

Every page should answer:

What?

Why?

How?

Next Step?

------------------------------------------------------------

# IMAGE STRATEGY

Real product photography.

No stock images where possible.

Consistent lighting.

Rounded corners.

Optimized images.

Lazy loading.

------------------------------------------------------------

# DOWNLOAD STRATEGY

PDF

Manual

Firmware

Drivers

Certificates

Technical Sheets

Grouped logically.

------------------------------------------------------------

# PERFORMANCE

Navigation must remain instant.

Search should feel immediate.

Dropdowns should open smoothly.

No layout shifts.

------------------------------------------------------------

# FINAL EXPERIENCE

Visitors should always know:

Where they are.

What they are viewing.

What they should do next.

Navigation should never become confusing,
even as the website grows.
